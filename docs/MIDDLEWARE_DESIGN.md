# 中间件设计

状态：草案
当前实现：请求日志、统一错误响应、JSON 响应辅助和配置加载基础能力已补充。

## 目标

中间件用于承载横切能力，例如请求日志、错误处理、CORS、限流等。后续增强应尽量保持
`HttpServer/` 的通用性，避免把短链接业务逻辑写入框架层。

## 优先级

### 1. 请求日志

记录请求方法、路径、状态码、耗时等基础信息，方便开发和排查问题。

状态：已完成基础版。

### 2. 统一错误响应

统一 API 错误格式，避免每个 handler 各自拼接错误响应。

状态：已完成基础版。当前框架默认错误响应使用 JSON 格式。

### 3. JSON 响应辅助工具

提供常用 JSON 响应构造方式，减少业务代码重复。

状态：已完成基础版。

### 4. 配置加载

为服务端口、日志、运行参数等提供统一配置入口。

当前暂定使用简单 `key=value` 文本格式，避免在当前阶段引入额外配置解析依赖。

状态：已完成基础版。

### 5. CORS 梳理

在已有 CORS 能力基础上明确配置和默认行为。

### 6. 限流

限流依赖配置、错误响应和可能的 Redis 支持，暂时后置。

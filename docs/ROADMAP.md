# 路线图

状态：草案
说明：本路线图会随着实现、验证结果和需求变化持续调整。

## 版本规划草案

版本号用于标记项目达到的可交付状态，不代表后续计划已经锁死。当前只明确近期版本，
远期版本保持草案状态，随着开发和学习过程持续调整。

| 版本 | 状态 | 目标能力 | 计划引入技术 |
| --- | --- | --- | --- |
| v0.1 | 当前 | 项目清理、公开文档基线、旧业务残留记录 | Markdown 文档、git 基线提交 |
| v0.2 | 计划 | 框架基础能力补强 | 请求日志、统一错误响应、JSON 响应辅助、配置加载 |
| v1.0 | 计划 | ShortLink 最小闭环 | HTTP API、短码生成、内存存储或临时存储、302 重定向 |
| v1.1 | 草案 | 持久化与缓存 | MySQL、Redis |
| v1.2 | 草案 | 工程化运行和质量验证 | Docker Compose、Nginx、测试、压测 |
| v2.0 | 暂缓 | 进阶业务和稳定性能力 | 访问统计、用户系统、消息队列、监控 |

当前策略：

- 近期版本只约束方向和边界，不提前锁死实现细节。
- v1.0 之前优先保持框架稳定和项目可 review。
- v1.1 之后的内容必须等 V1 最小闭环验证后再细化。
- 未实现能力不得在 README 或 docs 中描述为已完成。
